2. **Create tests/test_chat_stream.py** with four core tests:

   **Test approach:** Use `unittest.mock.patch` to mock `stream_printing_messages` in `src.agent.query`. The mock should be an async generator yielding a simple (Msg, False) then (Msg, True) pair. This simulates the streaming output without hitting any real API.

   The mock async generator pattern:
   ```python
   from agentscope.message import Msg

   async def mock_stream_generator(*args, **kwargs):
       # Yield a first chunk (not last)
       yield Msg(name="agentops", content="Hi", role="assistant"), False
       # Yield the final chunk (last=True)
       yield Msg(name="agentops", content="Hi there!", role="assistant"), True
   ```

   Then in each test:
   ```python
   with patch("src.agent.query.stream_printing_messages", side_effect=mock_stream_generator):
       response = client.post("/process", json=payload)
   ```

   **Test 1 -- test_process_returns_sse_stream:**
   - POST /process with valid payload.
   - Assert status_code == 200.
   - Assert "text/event-stream" in content-type header.
   - Assert response body contains "data:" lines (SSE format).

   **Test 2 -- test_stream_lifecycle_events:**
   - POST /process, collect all SSE events from response body.
   - Parse each "data: ..." line (skip "data: [DONE]").
   - Assert that the collected events contain at least one event with status "created" or "in_progress".
   - Assert that the collected events contain at least one event with status "completed".
   - This verifies D-04 (explicit lifecycle with start/delta/complete).
   - NOTE: The exact event shape depends on how agentscope-runtime wraps the (msg, last) tuples into SSE. The test should be flexible about exact field names but strict about the lifecycle pattern. Check for "status" field values across all parsed events.

   **Test 3 -- test_invalid_input_returns_http_error:**
   - POST /process with empty body `{}` -- the `input` field is required by AgentRequest.
   - Assert response status_code is 422 (FastAPI validation error), NOT 200 with SSE.
   - This confirms D-05: validation failures return HTTP errors before streaming.
   - Also test with `{"input": []}` (empty messages array) -- may or may not be 422 depending on how agentscope-runtime validates, but should not start streaming.

   **Test 4 -- test_repeated_requests_stable:**
   - Make two sequential POST /process calls with the same valid payload.
   - For each response, parse SSE events and verify:
     a. Response status_code == 200
     b. The SSE body contains events with "completed" status
     c. The SSE body ends with "data: [DONE]" or contains a completed response
   - This confirms D-08: repeat stability means lifecycle completes reliably, not identical text.

   **Important testing notes from research:**
   - The `content` field in messages must be `[{"type": "text", "text": "..."}]`, NOT a plain string (Pitfall 2 from research).
   - The SSE stream terminates with `data: [DONE]` -- test parsing must handle this sentinel.
   - Use `TestClient` (synchronous) for simplicity. For SSE, the response body will contain the full streamed output because TestClient reads the complete response. The response.text will have all "data:" lines.

   **If the SSE response is empty or malformed with TestClient:** This can happen if agentscope-runtime uses async streaming that doesn't work well with synchronous TestClient. If so, switch to `httpx.AsyncClient` with `ASGITransport` and use `pytest-asyncio`. Add `pytest-asyncio` to dev dependencies if needed. The pattern:
   ```python
   import pytest
   from httpx import AsyncClient, ASGITransport

   @pytest.mark.asyncio
   async def test_...(configured_env, clear_settings_cache, valid_payload):
       from src.main import app
       transport = ASGITransport(app=app)
       async with AsyncClient(transport=transport, base_url="http://test") as client:
           response = await client.post("/process", json=valid_payload)
   ```
   If this is needed, add `pytest-asyncio` to pyproject.toml dev dependencies.

   **If mocking stream_printing_messages is not sufficient:** If the AgentApp /process endpoint does additional processing that also needs mocking, mock at a higher level. The key principle is: tests must not make real API calls.
