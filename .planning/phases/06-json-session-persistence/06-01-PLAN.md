   This module provides a singleton JSONSession backend and session ID generation.

   ```python
   """Session persistence backend using agentscope-runtime JSONSession.

   Provides a singleton JSONSession instance for save/load operations
   and UUID-based session ID generation.

   Decisions: D-01 (JSONSession), D-04 (client-provided or auto-generated session_id),
   D-09 (flat sessions/ directory, configurable via SESSION_DIR).
   """

   import os
   import uuid
   import logging
   from pathlib import Path

   from agentscope.session import JSONSession

   from src.core.settings import get_settings

   logger = logging.getLogger(__name__)

   _session_backend: JSONSession | None = None


   def get_session_backend() -> JSONSession:
       """Return the singleton JSONSession backend instance.

       Creates the session directory at first access. The directory is
       configurable via SESSION_DIR env var (D-09), defaulting to ./sessions.
       """
       global _session_backend
       if _session_backend is None:
           settings = get_settings()
           session_dir = settings.SESSION_DIR
           _session_backend = JSONSession(save_dir=session_dir)
           logger.info("Session backend initialized: save_dir=%s", session_dir)
       return _session_backend


   def generate_session_id() -> str:
       """Generate a new unique session ID using UUID4 (D-04)."""
       return str(uuid.uuid4())


   def validate_session_id(session_id: str) -> bool:
       """Validate session_id format to prevent path traversal (T-6-01).

       Accepts only standard UUID format: 8-4-4-4-12 hexadecimal characters
       with hyphens, or plain alphanumeric strings. Rejects any path
       separators, dots, or special characters.
       """
       if not session_id or len(session_id) > 128:
           return False
       # Allow UUID format and simple alphanumeric/hyphen strings
       # Block path traversal characters: / \ . ..
       forbidden = ["/", "\\", ".", ".."]
       for ch in forbidden:
           if ch in session_id:
               return False
       return session_id.isprintable() and session_id.strip() == session_id
   ```

   Key design decisions:
   - `get_session_backend()` uses module-level singleton (Pattern 3 from RESEARCH.md)
   - `validate_session_id()` implements T-6-01 mitigation (path traversal prevention)
   - No TTL or cleanup (D-10)
   - UUID generation via `uuid.uuid4()` consistent with framework

**IMPORTANT pitfall from RESEARCH.md (Pitfall 1):**
`Runner.stream_query` normalizes session_id: `request.session_id = request.session_id or str(uuid.uuid4())`. This means the handler ALWAYS receives a non-empty session_id. The handler cannot distinguish "client sent session_id" from "runner auto-generated it" from the value alone. However, per D-05/D-08 and the Open Question #2 in RESEARCH.md, the correct behavior is: ALWAYS attempt session save. If the client didn't send a session_id, a new session file is created with the auto-generated ID. This aligns with D-04 ("When absent, a new session is created").

**CRITICAL pitfall (Pitfall 2):** Memory must be saved AFTER the `async for` loop completes, not during. The agent adds messages to memory during processing.

**CRITICAL pitfall (Pitfall 3):** Use the same keyword argument name `memory=` for both save and load calls.

   ```python
   @app.query(framework="agentscope")
   async def chat_query(self, msgs, request=None, **kwargs):
       """Handle chat queries with SSE streaming and optional session persistence.

       Creates a fresh agent per request (D-07). When session_id is present,
       loads prior memory state before agent creation and saves updated memory
       after streaming completes (D-08). When session_id is absent, behavior
       is identical to Phase 5 (backward compatible, D-05).

       Args:
           self: The AgentApp instance (passed by the decorator).
           msgs: The input messages from the request body.
           request: The raw AgentRequest object (carries session_id, agent_config).
           **kwargs: Additional keyword arguments from the framework.
       """
       # Extract per-request config override (D-01, D-04)
       agent_config = None
       if request and hasattr(request, "agent_config") and request.agent_config:
           agent_config = AgentConfig(**request.agent_config)
       config = resolve_effective_config(agent_config)

       # Session-aware memory (D-01, D-04, D-07)
       session_id = None
       session_backend = get_session_backend()
       memory = InMemoryMemory()

       if request and hasattr(request, "session_id") and request.session_id:
           raw_session_id = request.session_id
           if validate_session_id(raw_session_id):
               session_id = raw_session_id
               await session_backend.load_session_state(
                   session_id=session_id,
                   memory=memory,  # keyword name "memory" must match save (Pitfall 3)
               )
           # If session_id fails validation, skip session persistence (security: T-6-01)

       agent = ReActAgent(
           name="agentops",
           model=OpenAIChatModel(
               model_name=config["model_name"],
               api_key=config["api_key"],
               client_kwargs={"base_url": config["base_url"]},
               stream=True,
           ),
           sys_prompt="You are a helpful assistant.",
           formatter=OpenAIChatFormatter(),
           memory=memory,
           toolkit=toolkit,
       )
       agent.set_console_output_enabled(enabled=False)

       async for msg, last in stream_printing_messages(
           agents=[agent],
           coroutine_task=agent(msgs),
       ):
           yield msg, last

       # Save updated memory after streaming completes (D-08, Pitfall 2)
       if session_id:
           await session_backend.save_session_state(
               session_id=session_id,
               memory=agent.memory,  # keyword name "memory" must match load (Pitfall 3)
           )
   ```

   Key behavioral notes:
   - `session_id` comes from `request.session_id` which Runner already normalized to a UUID
   - When client sends no session_id, Runner generates one BUT `request.session_id` will be non-empty. The handler checks `request.session_id` -- it will always be present. Per RESEARCH.md Open Question #2 and D-04/D-05, this is correct: every request gets session persistence. Backward compatibility is preserved because new sessions start with empty memory (load_session_state with `allow_not_exist=True` skips when file doesn't exist).
   - When `validate_session_id` rejects the value, session persistence is silently skipped (security defense).

   WAIT -- re-reading D-05 more carefully: "When `session_id` is absent, behavior is identical to current stateless flow (backward compatible)." This means the handler MUST distinguish "client sent session_id" from "runner auto-generated it." Since the Runner always normalizes, we need a different approach.

   **Revised approach:** The client sends session_id in the request body. Before Runner normalization, if the client sent no session_id, it becomes None, then Runner sets it to UUID. To detect the original intent, check if the session FILE exists on disk:
   - If file exists --> this is a resume (client sent a valid existing session_id)
   - If file doesn't exist --> this is either a new session or stateless flow

   BUT this doesn't satisfy D-05 exactly. D-05 says "When session_id is absent, behavior is identical to current stateless flow." The cleanest approach: since Runner always sets session_id, we cannot distinguish from the request object alone. Per RESEARCH.md Open Question #2 recommendation: "Always attempt session save -- this is correct behavior." This means every request gets a session file. The key difference is that without an existing session file, `load_session_state` with `allow_not_exist=True` returns an empty memory -- which IS the same as the current stateless behavior.

   Final decision: Always use session persistence. This is simpler, aligns with the framework, and D-05 backward compatibility is maintained because new sessions start with empty memory (identical to current `InMemoryMemory()`).
