# Quick Task 260413-n8u: Session-scoped dynamic MCP bootstrap (`stdio` + `http`)

## Scope Guardrails
- Implement MCP only. Do not add dynamic skill loading, tool whitelist policy, or stateless HTTP MCP.
- Keep the current `/process` contract working for legacy callers that do not use bootstrap.
- Keep dynamic MCP isolated from the module-level shared toolkit.
- Prefer minimal invasive changes that replay existing defaults instead of rewriting the service shape.

## Decision Coverage
| Decision | Coverage |
|----------|----------|
| D-01 Dynamic MCP not in global shared toolkit | Tasks 1-2 move dynamic registration into a session runtime registry and keep `src/tools/__init__.py` global state legacy-only. |
| D-02 Bootstrap creates a session-scoped agent/toolkit | Tasks 1-2 create and store a bootstrapped `ReActAgent` + `Toolkit` per active session. |
| D-03 Static defaults may remain baseline capabilities | Tasks 1 and 3 keep legacy global defaults/startup MCP path intact for non-bootstrap callers. |
| D-04 HTTP config keeps `type="http"` and `transport=sse|streamable_http` | Task 1 defines the discriminated schema and client factory around those exact fields. |
| D-05 MCP resources are session-scoped, not per-message | Task 2 restricts MCP config to bootstrap and reuses the same runtime on `/process`. |
| D-06 One active session per pod | Tasks 1-2 enforce a single-active-session registry and return `409` on conflict. |
| D-07 Close via explicit shutdown or pod teardown | Tasks 2-3 add a shutdown endpoint and lifespan teardown hook. |
| D-08 HTTP MCP uses stateful semantics | Tasks 1 and 3 use `HttpStatefulClient` only and explicitly reject `HttpStatelessClient`. |
| D-09 Any MCP init/register failure aborts bootstrap | Tasks 1-2 use all-or-nothing bootstrap with rollback and clear errors. |
| D-10 No partial availability / optional flags | Task 1 omits optional-MCP semantics and uses `namesake_strategy="raise"` plus fail-fast bootstrap. |

**Dependency order:** Task 1 -> Task 2 -> Task 3

<objective>
Evolve the current per-request agent creation path into an opt-in session bootstrap flow that can accept MCP server definitions from request JSON, instantiate a session-scoped agent/toolkit once, and reuse it for later chat turns.

Purpose: Support dynamic MCP experiments without contaminating other sessions or rewriting the existing service architecture.
Output: Bootstrap/shutdown API contracts, session runtime registry plus client factory, `/process` runtime reuse path, and tests/demo that prove `stdio`/`http` handling and cleanup semantics.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| bootstrap JSON -> session runtime builder | Untrusted client input controls which MCP clients are created |
| server -> stdio subprocess | The service launches external processes from request-provided config |
| server -> remote HTTP MCP | The service opens authenticated/stateful network sessions to external MCP endpoints |
| session registry -> `/process` reuse | A live in-memory runtime is reused across later requests |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-260413-n8u-01 | T | `SessionBootstrapRequest` / client factory | mitigate | Validate with discriminated Pydantic models, `extra="forbid"`, fixed `type` values, and direct constructor args (no shell strings, no ad-hoc transport abstraction). |
| T-260413-n8u-02 | E | stdio MCP bootstrap | mitigate | Pass `command` and `args` as structured fields to `StdIOStatefulClient`; do not concatenate shell commands or register dynamic MCP into the global toolkit. |
| T-260413-n8u-03 | I | HTTP MCP headers / error logging | mitigate | Redact headers and secrets in logs/errors; log server name/type/transport only, not header values. |
| T-260413-n8u-04 | D | bootstrap failure / leaked clients | mitigate | Use all-or-nothing bootstrap with LIFO rollback on failure, plus explicit shutdown and teardown close paths. |
| T-260413-n8u-05 | E | cross-session contamination | mitigate | Enforce one active session per pod and resolve runtimes via session-scoped registry instead of the shared `toolkit` singleton. |
