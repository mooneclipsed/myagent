# Quick Task Plan: Refactor Tool Registration Wiring

## Objective

Extract tool registration and registry-management helpers into a dedicated module under `src/tools/` while preserving the current public import surface and runtime behavior for both the legacy shared-toolkit path and the session bootstrap path.

## Plan 1: Extract Registry Logic with Behavior Parity

### Task 1: Create a dedicated registry module
- **files**: `src/tools/registry.py`
- **action**: Move the registry/registration responsibilities out of `src/tools/__init__.py` into `src/tools/registry.py`. The new module should own `_example_skill_dir`, `register_default_tools()`, `register_legacy_example_skill_support()`, `create_base_toolkit()`, `TOOL_REGISTRY`, `ToolRegistryError`, and `register_configured_tools()`. Keep function names, signatures, tool names, group names, summary generation, and fail-fast unknown-tool validation behavior unchanged so the current bootstrap/name-based registry flow remains behaviorally identical. Do not move the shared singleton `toolkit` or legacy startup `_mcp_clients` list into this module.
- **verify**: `uv run pytest tests/test_tools.py::TestToolRegistry -x -v`
- **done**: `src/tools/registry.py` becomes the single home for tool registration and name-based registry logic without changing observable behavior.

### Task 2: Reduce `src.tools` to package exports and shared legacy state
- **files**: `src/tools/__init__.py`
- **action**: Replace the inlined registry/registration implementation in `src/tools/__init__.py` with imports/re-exports from `src.tools.registry`. Keep only the package-level objects that truly belong in the package entrypoint for current wiring: `toolkit = create_base_toolkit(include_legacy_example_skill_support=True)` and `_mcp_clients`. Preserve existing imports from `src.tools` so `src.agent.session_runtime`, `src.agent.query`, `src.app.lifespan`, and current tests continue to work without call-site churn. Avoid editing those consumers unless import resolution makes it strictly necessary, because the requested scope is a refactor with unchanged behavior.
- **verify**: `uv run pytest tests/test_tools.py::TestToolRegistration tests/test_skill_runtime.py -x -v`
- **done**: `src/tools/__init__.py` is a thin package entrypoint that still exposes the same public API and legacy shared-toolkit behavior.

### Task 3: Add extraction-safe regression coverage
- **files**: `tests/test_tools.py`
- **action**: Add focused regression assertions that the moved helpers can be imported directly from `src.tools.registry` and are still re-exported from `src.tools`. Reuse the existing registry tests instead of broad test rewrites. Rely on the existing bootstrap integration tests in `tests/test_session_bootstrap.py` to confirm that tool summaries, empty-tool bootstraps, and unknown-tool validation behavior remain unchanged after the extraction.
- **verify**: `uv run pytest tests/test_tools.py tests/test_session_bootstrap.py tests/test_skill_runtime.py -x -v`
- **done**: Tests lock in the new module boundary and prove that legacy/shared-toolkit and bootstrap behaviors are unchanged.

## Success Criteria

- Tool registration and name-based registry management no longer live in `src/tools/__init__.py`.
- `src/tools/registry.py` is the canonical home for toolkit construction and configured-tool registration helpers.
- Existing imports from `src.tools` continue to work unchanged.
- Legacy `/process` shared-toolkit behavior remains unchanged.
- Bootstrapped session tool registration, tool summaries, and unknown-tool validation remain unchanged.
- Focused regression tests pass with minimal, localized edits.
